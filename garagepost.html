Garage Nonsense

Entering in cars to be fixed, then pretending to fix them

Garage Project Pt. 1

The Garage task was split up into two separate challenges. The first challenge was to create a virtual garage with which you can check in and out cars, log faults, display the contents of the garage, and calculate the cost of fixing the cars which had faults.

For the first challenge, I had a large number of buttons, each mapped to its own function, as well as multiple text boxes. While this all works, it looks clunky.

Output is displayed below the buttons - border automatically expands

Garage Project Pt. 2

For the second Garage challenge, I had to compact all of the buttons and textboxes from the previous challenge into a single textbox and button (simulating a console log).

The JavaScript first reads the textbox entry as an array, with the first word being in array slot [0], the second word in slot [1], etc. It then interprets the slot [0] entry as the command, and the following slots ([1],[2],etc.) as the data being manipulated. For example, typing "input BMW X5 XYXYXYX 2015" would add a new car to an array of cars with the data "brand:BMW; model:X5; registration:XYXYXYX; year:2015".

There are multiple ways to interact with the JavaScript, depending on what the first word you type is
